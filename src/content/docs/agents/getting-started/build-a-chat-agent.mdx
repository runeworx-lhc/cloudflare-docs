A complete guide to building a streaming AI chat agent with tool integration using Cloudflare Workers and the Agents framework.

> **Complete example**: See the full implementation in the [agents-starter repository](https://github.com/cloudflare/agents-starter) on GitHub.

## Prerequisites

- Node.js 18+
- npm or yarn
- Cloudflare account
- OpenAI API key

## Step 1: Set up the project

```bash
# Create the project using the Cloudflare agents starter template
npm create cloudflare@latest chat-agent -- --template=cloudflare/agents-starter

# Navigate to the project directory
cd chat-agent

# Install dependencies
npm install
```

## Step 2: Configure your environment

You have two options for setting up your OpenAI API key:

**Option A: Using wrangler.jsonc (recommended for production)**

```json
{
	"compatibility_date": "2024-09-23",
	"compatibility_flags": ["nodejs_compat"],
	"vars": {
		"OPENAI_API_KEY": "your-openai-api-key-here"
	},
	"define": {
		"process.env.NODE_ENV": "\"production\""
	}
}
```

**Option B: Using .dev.vars (recommended for development)**

```bash
echo "OPENAI_API_KEY=your-key-here" > .dev.vars

```

**⚠️ Security Best Practice**: For production deployments, use Cloudflare secrets instead of putting API keys in `wrangler.jsonc`:

```bash
# Set the OpenAI API key as a secret (recommended for production)
wrangler secret put OPENAI_API_KEY
```

## Step 3: Test the agent locally

```bash
# Start the development server
npm start

# The server will be available at http://localhost:5174
```

Open your browser to `http://localhost:5174` and start chatting!

#### Tools (`src/tools.ts`)

Define what actions your agent can perform:

<TypeScriptExample>

```ts
import { tool } from "ai";
import { z } from "zod";

// Automatic execution (no confirmation needed)
const getLocalTime = tool({
	description: "get the local time for a specified location",
	parameters: z.object({ location: z.string() }),
	execute: async ({ location }) => {
		return "10am";
	},
});

// Requires confirmation (no execute function)
const getWeatherInformation = tool({
	description: "show the weather in a given city to the user",
	parameters: z.object({ city: z.string() }),
	// Omitting execute function makes this tool require human confirmation
});
```

</TypeScriptExample>

## Customization

### Adding custom tools

**Simple Tool (Automatic Execution)**

<TypeScriptExample>

```ts
// In src/tools.ts
const getWeather = tool({
	description: "Get current weather for a location",
	parameters: z.object({
		location: z.string().describe("The city name"),
	}),
	execute: async ({ location }) => {
		// Call weather API
		return `The weather in ${location} is sunny and 72°F`;
	},
});

// Add to exports
export const tools = {
	// ... existing tools
	getWeather,
};
```

</TypeScriptExample>

**Confirmation-Required Tool**

<TypeScriptExample>

```ts
// In src/tools.ts
const sensitiveAction = tool({
	description: "Perform a sensitive action",
	parameters: z.object({
		action: z.string().describe("The action to perform"),
		reason: z.string().describe("Reason for the action"),
	}),
	// Omitting execute function makes this tool require human confirmation
});

// Add to exports
export const tools = {
	// ... existing tools
	sensitiveAction,
};

// Add execution function
export const executions = {
	// ... existing executions
	sensitiveAction: async ({ action, reason }) => {
		return `Action "${action}" completed successfully. Reason: ${reason}`;
	},
};
```

</TypeScriptExample>

## Troubleshooting

### Common issues

**"OPENAI_API_KEY is not set"**

- Make sure you've set the API key in `.dev.vars` or `wrangler.jsonc`
- For production, use `wrangler secret put OPENAI_API_KEY`

**"npm run dev" not found**

- Use `npm start` instead (the script is named "start")

**Server not accessible**

- Check that the server is running on port 5174 (not 8787)
- Look for the correct URL in the terminal output

**Tool execution errors**

- Make sure tools are properly exported in `tools` object
- Check that confirmation-required tools have execution functions in `executions`

### Development tips

- Use the browser's developer tools to see detailed error messages
- Check the terminal output for server logs
- The debug mode in the UI shows detailed message processing
